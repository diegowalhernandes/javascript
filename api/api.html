O que é API

  • Application Programming Interface (Interface de Programação de Aplicações);
  • Simplifica desenvolvimento, não necessita de telas;
  • Fácil integração com sistemas, apenas dando respostas;
  • Padrão de desenvolvimento, permitindo a troca de APIs;
  • Perfromace por causa das requisições e respostas rápidas;
  • Exemplo prático: cotação de frete;

    • Tipos de API

  • API pública: disponibilizada paa qualquer um acessar, sem necessidade de autenticação;
  • API privada: quando uma API é criada para utilização entre sistemas de uma empresa, requer autenticação;
  • API de terceiros: quando a API é um produto, podemos ser integrada em um sistema, requer autenticação;

    • Comunicação com API

  • Normalmente feita através de XML ou JSON;
  • Devido a simplicidade destes tipos de dado;
  • As requisições e respostas são feitas através do HTTP;
  • Onde podemos seguir um padrão, para criação de APIs robustas e que serão de fácil utilização;

    • Verbos do HTTP

  • Alterando os verbos podemos realizar requisições diferentes, simplificando a construção da nossa API e seus endpoints;
  • GET: verbo para resgatar algum recurso;
  • POST: com o POST queremos enviar dados a API;
  • DELETE: Para remover algum registro do banco;
  • PUT: atualização de registros;
  • PATCH: atualização parcial de registros;

    • O que é endpoint

  • Endpoint é uma URL que acessamos da API para extrair uma resposta;
  • Exemplo: /users/1 - GET
  • Mesmo endpoints com verbos diferentes podem ter resultados diferentes
  • Esta é uma das técnicas para reduzir a complexidade e aplicar o padrão REST;

    • O que é REST?

  • RESTé um padrão/conceito de arquitetura ligado a APIs e o protocolo HTTP;
  • Podemos implementar este padrão, que é composto de 6 partes para tornar nossa API em RESTful
  • Eles são: Uniformidade, Desacoplação, Stateless, Cache, Arquitetura de Camadas, Code on demand;

    • Conhecendo o Postman

  • Postman é um softaware que permite testar as APIs que criamos;
  • Pode,os inserir um endpoint, método HTTP, inserir dados no corpo da requisição e muito mais;
  • Ele é muito utilizado na indústria de software
